# 分析项目并给出完整部署脚本

## Goal

分析 `homework-agent` 项目的实际结构、运行入口、依赖和配置方式，交付一个完整、可执行、失败显式暴露的部署脚本。

## Scope

- 识别 backend、agent、frontend 的构建和运行方式。
- 识别现有 Makefile、README、env 示例和源码配置约束。
- 新增或修正部署脚本，不引入 mock、静默降级或掩盖失败的 fallback。
- 通过语法检查、构建或测试验证脚本和项目关键路径。

## Out Of Scope

- 不重构业务代码。
- 不回滚用户已有未提交改动。
- 不引入容器化或进程管理器，除非现有项目已明确依赖。

## Acceptance Criteria

- 部署脚本可从干净环境执行依赖安装、配置校验、构建和服务启动/停止/状态查看。
- 脚本明确暴露缺失依赖、缺失环境变量和端口占用等失败。
- 文档或最终答复说明脚本路径、命令和必要配置。
- 自动验证命令执行并记录结果。
